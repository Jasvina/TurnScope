# Screenshots guide

Use this when adding or refreshing README visuals, demo images, or release screenshots.

## What to capture

- the session timeline
- the process tree or shell lane
- the diff view
- the raw event inspector
- the subagent graph
- the collector index or bundle loader, if it helps explain the workflow

## Capture rules

- Use sample data only.
- Remove secrets, tokens, hostnames, user names, and customer data.
- Prefer wide browser captures that keep labels readable at 100% zoom.
- Keep one screenshot focused on one idea.
- Store new assets under `docs/assets/` with descriptive names.

## Review checklist

- Text is legible.
- Important UI labels are visible without scrolling.
- The image still matches the current UI.
- Nothing in the screenshot could create a privacy or security issue if the file were public.
